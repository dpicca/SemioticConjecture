Output schema¶

This page documents the CSV files written by the CLI pipeline. It focuses on what each file contains, how it is produced, and the exact column schemas.

Note : General CSV conventions used across all outputs - Encoding: UTF-8, newline terminator \n - Floats: formatted to 6 decimals - Column order: first-seen order is preserved by the writer

Overview¶

File Purpose Emitted by Key columns
cleaned_input.csv Validated copy of the input corpus semiconj.corpus.excerpts_to_rows → write_csv id, text, meta_*
S_metrics.csv Semiotic amplitude components per text and aggregate S semiconj.cli.compute_S_components, semiconj.decodability.aggregate_S semantics, intertextuality, figures, lexicon, pos, codeswitch, S
D_intr.csv Intrinsic decodability components and aggregate semiconj.semiotic.parser.compute_d_intr structural_precision, convergence, coherence, entropy, D_intr
Omega.csv Corpus-level interpretive homogeneity per context semiconj.surrogates.community.omega_and_rho context, Omega
Rho.csv Corpus-level contextual rigidity/constraint per context semiconj.surrogates.community.omega_and_rho context, Rho
D_effective.csv Effective decodability per text and context semiconj.decodability.effective_decodability id, context, D
Frontier.csv Non‑parametric frontier points (95th percentile of X=S·D per S bin) semiconj.frontier.estimate_frontier S_bin, k95
Analyses.csv Extra analytical summaries semiconj.analysis.* metric, value

cleaned_input.csv¶

  • Purpose: cleaned copy of the input corpus after validation
  • Emitted by: semiconj.corpus.excerpts_to_rows and semiconj.corpus.write_csv

Columns

Column Type Range/Domain Description
id string — Unique identifier, first occurrence kept on duplicates
text string — Raw text excerpt
meta_* string optional Any additional input columns are kept; non meta_ names are prefixed

Notes - Rows failing validation (missing id/text, duplicate id, too short) are removed prior to writing.

S_metrics.csv¶

  • Purpose: per‑text semiotic amplitude components and aggregate
  • Emitted by: semiconj.cli.compute_S_components + semiconj.decodability.aggregate_S via semiconj.reporting.save_metrics

Columns

Column Type Range/Domain Description
id string — Record identifier
semantics float [0,1] Avg of sentence embedding dispersion and domain coverage
intertextuality float [0,1] Avg of NER-like coverage and domain coverage
figures float [0,1] Metaphor/irony proxy score
lexicon float [0,1] MTLD (normalized), Yule’s K (mapped), senses/lemma (norm), bigram entropy (avg)
pos float [0,1] POS-tag distribution entropy (UPOS-normalized)
codeswitch float [0,1] Code‑switching index
S float [0,1] Weighted aggregate using semiconj.config.DEFAULT_WEIGHTS

D_intr.csv¶

  • Purpose: intrinsic decodability components and aggregate per text
  • Emitted by: semiconj.semiotic.parser.compute_d_intr via semiconj.reporting.save_metrics

Columns

Column Type Range/Domain Description
id string — Record identifier
structural_precision float [0,1] Triads per sentence (clipped)
convergence float [0,1] Mean pairwise cosine among interpretants
coherence float [0,1] 1 − dispersion across sentences
entropy float [0,1] 1 − mean similarity among interpretants (penalty)
D_intr float [0,1] Aggregated intrinsic decodability

Omega.csv¶

  • Purpose: corpus-level interpretive homogeneity per context
  • Emitted by: semiconj.surrogates.community.omega_and_rho via semiconj.reporting.save_metrics

Columns

Column Type Range/Domain Description
context string typically {C_open, C_medium, C_rigid} Context identifier (configurable)
Omega float [0,1] Interpretive homogeneity of the community

Rho.csv¶

  • Purpose: corpus-level contextual rigidity/constraint per context
  • Emitted by: semiconj.surrogates.community.omega_and_rho via semiconj.reporting.save_metrics

Columns

Column Type Range/Domain Description
context string typically {C_open, C_medium, C_rigid} Context identifier (configurable)
Rho float [0,1] Rigidity/constraint of the context

D_effective.csv¶

  • Purpose: effective decodability per text and context
  • Emitted by: semiconj.decodability.effective_decodability via semiconj.reporting.save_metrics

Columns

Column Type Range/Domain Description
id string — Record identifier
context string — Context name used to compute D
D float [0,1] Effective decodability = Ω · ρ · D_intr

Frontier.csv¶

  • Purpose: non‑parametric frontier k(H,C) estimated as the 95th percentile of X = S·D within S bins
  • Emitted by: semiconj.frontier.estimate_frontier via semiconj.reporting.save_metrics

Columns

Column Type Range/Domain Description
S_bin float [0,1] Representative S value (bin center)
k95 float [0,1] 95th percentile of X = S·D in that bin

Analyses.csv¶

  • Purpose: extra analytical summaries (may grow over time)
  • Emitted by: semiconj.analysis.* via semiconj.reporting.save_metrics

Columns

Column Type Range/Domain Description
metric string — Metric name (e.g., Kendall_tau_S_vs_Dintr)
value float — Metric value
